2 Introduction
Sur les réseaux numériques, une fausse information peut être publiée par un site peu
fiable, citée par d’autres sites, puis par d’autres encore, se propageant ainsi à grande
vitesse avant d’être démentie. Ce phénomène s’appelle la désinformation.
Ce projet a pour objectif de concevoir et d’implémenter, en langage C, une structure
de données permettant de modéliser un réseau de citations entre articles, d’analyser la
propagation d’une fake news, et de détecter automatiquement les articles suspects.
Le réseau d’information est modélisé sous forme d’un graphe orienté où chaque
nœud représente un article publié sur le web, et chaque arête orientée A → B signifie
que l’article A cite l’article B.
Ce graphe est orienté : si l’article A cite l’article B, cela ne signifie pas que B cite A
en retour.
Définition d’un article — TDA ELEMENT
Dans ce projet, le TDA ELEMENT vu en cours est réalisé par un article d’infor-
mation. Vous devez gérer ELTARTICLE.H et ELTARTICLE.C.
Un article possède :
— un identifiant unique (id),
— un titre (titre),
— le nom du site ou de la page qui l’a publié (source),
— un score de fiabilité entre 0 et 100 (score_fiabilite) : 0 signifie que l’article est
une fake news certaine, 100 signifie qu’il est totalement fiable,
— sa date de publication : jour, mois, annee, heure, minute.
/***************************************************
* Fichier : ELTARTICLE.H
* Realisation du TDA ELEMENT par un article
***************************************************/
typedef struct {
int id;
char titre[100];
char source[50];
int score_fiabilite;
int jour, mois, annee;
int heure, minute;
} articleStruct, *ELEMENT;
#define ELEMENT_VIDE NULL
Par exemple :
— L’article “Vaccin_cache_dangereux” publié par la page Facebook page_TunisieActu
le 15/03/2025 à 09h00, avec un score de fiabilité de 8, est très suspect.
— Le communiqué officiel “Communique_officiel” publié par le site du Ministère de la
Santé tunisien le 15/03/2025 à 07h00, avec un score de 90, est fiable.

Structure des relations entre articles
Les relations de citation entre articles sont représentées par une liste d’adjacence :
pour chaque article i, on maintient la liste des articles qu’il cite. Les éléments de cette
liste sont de type ELEMENT (des articles, au sens du TDA ELEMENT vu en cours).
typedef struct {
int V;
ELEMENT *articles; /* tableau des articles [0..V-1] */
LISTE *adjList; /* adjList[i] = LISTE des articles */
/* cites par l’article i */
int *degre_in; /* degre_in[i] = nb d’articles */
/* qui citent l’article i */
} *grapheReseau;

3 Format du Fichier d’Entrée
Le graphe est chargé depuis un fichier texte structuré comme suit. Chaque ligne com-
mence par une lettre indiquant son type : A pour un article, C pour une citation. Les
lignes commençant par # sont des commentaires ignorés.
# Format : A id titre source score jour mois annee heure minute
# C id_src id_dest
A 0 "La_verite_sur_le_vaccin_alerte_secret_choc" page_TunisieActu 8 15 3 2025 9 0
A 1 "Etude_exclusif_urgent_OMS_choc_complot" Kapitalis 45 15 3 2025 8 0
A 2 "Partagez_avant_suppression_urgent_choc_alerte" TunisieNumerique 5 15 3 2025 11 0
A 3 "Communique_officiel_MinSante_MinistereSante" MinistereSante 90 15 3 2025 7 0
A 4 "Wake_up_complot_interdit_censure_vaccin" RTArabic 15 15 3 2025 14 0
C 0 1 # Vaccin_cache cite Etude_OMS
C 0 3 # Vaccin_cache cite Communique
C 1 3 # Etude_OMS cite Communique
C 2 0 # Partagez_avant cite Vaccin_cache
C 4 0 # Alerte_vaccin cite Vaccin_cache
C 4 2 # Alerte_vaccin cite Partagez_avant
Règles importantes :
— Chaque article est décrit sur une seule ligne commençant par A.
— Les articles doivent être déclarés avant leurs citations.
— Les identifiants sont des entiers consécutifs à partir de 0.
— Le titre est entre guillemets et ne contient pas d’espaces.
— Le score est un entier entre 0 et 100.
— L’heure et la minute sont des entiers (ex : 9 pour 09h00, 30 pour 30 minutes).
Génération du fichier de test : Pour tester votre projet avec un réseau plus grand,
vous pouvez demander à un outil d’IA générative (ChatGPT, Gemini, etc.) de générer un
fichier de données en respectant strictement ce format. Par exemple :
"Génère un fichier de données pour un réseau de 10 articles
sur la désinformation en Tunisie, au format suivant :
A id titre source score jour mois annee heure minute
C src dest
avec des dates cohérentes (on ne cite que des articles
déjà publiés), des scores variés entre 5 et 95,
et au moins 2 articles qui se citent mutuellement."

Vous vérifierez que le fichier généré respecte bien le format avant de l’utiliser.

4. Simulation de propagation — Parcours en Largeur (BFS)
Breadth-First Search (BFS), ou Parcours en Largeur, est un algorithme clas-
sique d’exploration de graphe. L’idée est simple : imaginez qu’une rumeur se propage
sur les réseaux sociaux. D’abord, tous les sites qui citent directement la source ap-
prennent la nouvelle (niveau 1). Ensuite, les sites qui citent ces sites apprennent la
nouvelle (niveau 2), et ainsi de suite. Le BFS explore exactement dans cet ordre, ni-
veau par niveau, ce qui permet de savoir à quelle distance (en nombre de citations)
un article se trouve de la source.
On utilise une file FIFO (premier entré, premier sorti) pour implémenter le BFS : on
place la source dans la file, puis on retire un article, on enfile ses voisins non encore
visités, et on recommence jusqu’à ce que la file soit vide. On réalise cette file avec le
TDA LISTE vu en cours :
— enfiler (ajouter en queue) : inserer(file, e, listeTaille(file)+1)
— défiler (retirer en tête) : recuperer(file, 1) puis supprimer(file, 1)
Remarque : si vous n’êtes pas encore familier avec le BFS, vous pouvez le rechercher
dans la littérature algorithmique ou demander à un outil d’IA de vous l’expliquer
avant de l’implémenter. L’important est que vous compreniez le principe et que vous
l’implémentiez vous-même en utilisant les primitives du TDA LISTE.
/* Simule la propagation depuis l’article idSrc : */
/* affiche niveau par niveau les articles atteints. */
/* Un article deja atteint n’est pas traite deux fois. */

5. Détection naïve de fausses informations

mots suspects fréquemment utilisés dans les fake news :

#define NB_SUSPECTS 8

"ce qu on vous cache", "la verite sur",
"les medias ne disent pas","wake up"
};

"alerte", "urgent", "exclusif", "censure",
"complot", "secret", "interdit", "choc"

/* Calcule un score de suspicion pour un article : */
/* +40 pts si le titre contient un element de */
/* BASE_FAKES (recherche insensible a la casse) */
/* +10 pts par mot de MOTS_SUSPECTS trouve dans */
/* le titre */
/* Met a jour art->score_fiabilite = */
/* max(0, 100 - score_suspicion) */
/* Retourne le score de suspicion calcule */
int analyserArticle(ELEMENT art);
/* Applique analyserArticle() a tous les articles */
